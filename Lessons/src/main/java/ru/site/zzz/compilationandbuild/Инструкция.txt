В папке есть 2 java файла с форматом *.java. Там код, который можно прочитать через IDE или notepad++.

Если открыть cmd или bash и попробовать запустить Start.java через команду java Start или java Start.java, то не запустится. Будет ошибка. Не находит мэин класс для загрузки. *.java нельзя запустить. Нужно запускать через *.class.

В консоли написать javac Start.java. Тогда будет создано сразу 2 файла *.class, так как в Start.java указан еще дополнительный, второй, файл *.java, поэтому будет создано 2 файла *.class. Если бы в Start.java не был бы прописан второй файл, то он не был бы получен *.class. Был бы получен 1 файл *.class. Если компилировать Score.java (второй файл), то будет Score.class - без первого файла Start.class.

Написать команду javac Start.java и будет получено 2 новых файла с форматом *.class

Далее написать команду java Start. Запустить файл, в котором формат *.class. В папке создастся scores.txt. Там будут записи. Важно: в папке есть Score.class (значек ТХТ) и score.txt (значек ТХТ) - не перепутать.

Чтобы автоматизировать создание файлов *.class, нужно создать батник.
Название файла: Start compilation.bat
Внутри файла 1 строка: javac Start.java
Запускается батник, создается 2 файла *.class.

Чтобы запустить Start.class. нужно запустить второй батник.
Название файла: Start program.bat
Внутри файла 1 строка: java Start
В папке создастся scores.txt. Там будут записи.

Чтобы собрать в jar-архив нужно написать команду jar -cf Program.jar *.class
Эта команда соберет все файлы с расширением *.class.
Jar-файл будет называться Program.jar
Далее нужно извлечь из jar-файла MANIFEST.MF файл (не папку, файл). добавить третью строку Main-class: Start и после нее добавить интэр - 2 шт, хотя раньше 1 был. Сохранить. И этот файл обратно в архив положить с заменой.

Всё, что написано в инструкции работает.
